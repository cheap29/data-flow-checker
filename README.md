# 要件設計サポートアプリ

お客様ヒアリングから業務要件定義までをAIがサポートするWebアプリです。  
ヒアリング内容を整理して確認事項・提案を出し、合意した内容を業務要件レビューに引き渡すフローを一貫して管理できます。

## 主な機能

### ヒアリング整理タブ

- **2つの入力モード**
  - 整理済みテキスト：現状（As-Is）となりたい姿（To-Be）を入力
  - メモ貼り付け：箇条書きメモをそのまま貼るだけでAIが自動整理
- **確認事項のステータス管理**（未対応 / 確認中 / 確認済 / スコープ外）
  - セグメント式プログレスバーで進捗を可視化
- **提案のステータス管理**（提案中 / 採用 / 見送り）
  - クライアントと合意した提案だけを「採用」にマーク
  - 採用済みの提案が揃ったら業務要件レビューへ転記ボタンが出現
- **効率化の見込み**：改善で期待できる効果を定量的に提示

### 業務要件レビュータブ

- **確認が必要なこと**（ステータス管理）
  - 抜けていそうな仕様・矛盾・リスクを重要度（高 / 中 / 低）付きで表示
  - 現場に確認したい質問を生成
- **確定していること**
  - 主要データオブジェクト、影響を受ける画面・機能、利用者別の困りごと
- **テスト観点**（正常系・異常系・境界値）：チェックボックスで消し込み
- **データの流れ図**
  - アクター（ユーザー / フロントエンド / バックエンド / DB / 通知など）別タイムライン表示
  - 詳細表示 / シンプル表示の切替
- PCでは2カラムレイアウト（左：チェックリスト / 右：データフロー図）、モバイルは1カラム
- Markdownコピー機能

## セットアップ

### 1. 依存パッケージのインストール

```bash
npm install
```

### 2. 環境変数の設定

`.env.local` ファイルをプロジェクトルートに作成し、OpenAI APIキーを設定してください。

```env
OPENAI_API_KEY=your_openai_api_key_here
```

APIキーは [OpenAI Platform](https://platform.openai.com/api-keys) から発行できます。  
**推奨：** キー作成時は権限を **Restricted（Model capabilities のみ）** に設定してください。

### 3. 起動

```bash
npm run dev
```

ブラウザで `http://localhost:3000` を開いてください。

## 使い方

### ヒアリング整理

1. 「ヒアリング整理」タブを開く
2. 入力モードを選ぶ
   - **整理済みテキスト**：現状・なりたい姿をそれぞれ入力
   - **メモ貼り付け**：ヒアリング中に取ったメモをそのまま貼り付け
3. 「ヒアリングを整理する」をクリック
4. 確認事項のステータスを管理しながらヒアリングを進める
5. クライアントと合意できた提案を「採用」にマーク
6. 「採用した提案を業務要件レビューで分析する」ボタンで次のステップへ

### 業務要件レビュー

1. 「業務要件レビュー」タブを開く（ヒアリング整理から引き継ぐか、直接入力）
2. 「チェックする」ボタンをクリック
3. 右パネルでデータの流れを確認しながら、左パネルの確認事項をステータス管理
4. 「Markdownコピー」で議事録・チケットに貼り付け

- APIキーはサーバーサイド（Next.js Route Handler）のみで使用し、クライアントには露出しません
